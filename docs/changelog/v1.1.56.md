# v1.1.56：Task 渲染修复、Windows 打开应用更顺手

**发布日期**：2026年02月11日

---

## 一句话总结

这是一个桌面端体验修复版本：Task 工具渲染问题修复、Windows 现在支持用可执行程序打开应用、切换会话时侧边栏不再自动收起。

---

## 问题修复：Task 工具在桌面端渲染异常

### 之前的问题

在桌面应用里，Task 工具相关内容存在渲染异常，影响阅读和操作。

### 现在的情况

v1.1.56 已修复渲染问题，Task 执行过程显示更稳定。

### 如何使用？

👉 **完整教程**：[Agent 权限与安全](/5-advanced/02c-agent-permissions)

👉 **官方文档**：[Commands / Subtask](https://opencode.ai/docs/commands#subtask)

**快速体验**：

1. 在桌面端发起需要子任务的请求。
2. 观察 Task 输出和步骤渲染是否正常。
3. 在同一会话中重复触发，确认显示稳定。

---

## 新功能：Windows 支持通过可执行程序打开应用

### 这次更新了什么？

桌面端新增了在 Windows 中通过可执行程序打开应用的能力。

### 这对你有什么用？

本地工具联动更直接，不需要绕很多手动步骤，尤其在企业环境里更实用。

### 如何使用？

👉 **完整教程**：[2.1 界面与操作](/2-daily/01-interface)

**快速体验**：

1. 在 Windows 桌面端打开相关入口。
2. 选择目标应用或文件并触发打开。
3. 确认系统使用预期可执行程序启动。

---

## 改进：切换会话不再自动收起侧边栏

### 这次更新了什么？

切换会话时，侧边栏状态会保留，不会被自动关闭。

### 这对你有什么用？

多会话来回切换时操作更连贯，尤其适合需要频繁对比上下文的工作流。

---

## 要不要升级？

| 情况 | 建议 |
|------|------|
| 你主要用桌面端 | ✅ 建议升级 |
| 你在 Windows 上工作 | ✅ 建议升级 |
| 你经常用 Task/子任务 | ✅ 建议升级 |
| 用得好好的 | 可以等下次 |

**升级命令**：

```bash
opencode upgrade
```
