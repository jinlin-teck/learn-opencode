# v1.1.55：超额提示更友好、事件内存泄漏修复

**发布日期**：2026年02月11日

---

## 一句话总结

这次更新重点是「稳定和可预期」：免费额度超限时给出更清晰提示，修复事件抓取内存泄漏，并在桌面端禁用终端透明效果来减少显示问题。

---

## 新功能：免费额度超限提示更清晰

### 这次更新了什么？

当免费使用额度超限时，OpenCode 会显示更明确、更可操作的提示信息。

### 这对你有什么用？

你不会再卡在「为什么突然不能继续」的状态里，能更快判断是额度问题还是配置问题。

### 如何使用？

👉 **完整教程**：[免费模型（OpenCode Zen）](/1-start/04a-free-models)

👉 **官方文档**：[Zen](https://opencode.ai/docs/zen)

**快速体验**：

1. 使用免费模型持续对话到接近额度上限。
2. 观察提示文案是否给出明确原因。
3. 按提示切换模型或等待额度刷新。

---

## 问题修复：事件抓取内存泄漏

### 之前的问题

事件抓取路径存在内存泄漏风险，长时间运行后可能出现卡顿或占用升高。

### 现在的情况

v1.1.55 已修复相关泄漏点，长会话稳定性更好。

---

## 改进：桌面端禁用终端透明

### 这次更新了什么？

桌面端默认关闭终端透明效果。

### 这对你有什么用？

在不同系统主题和显卡驱动下，终端显示更一致，减少对比度不清和渲染异常。

👉 **完整教程**：[调试与诊断](/5-advanced/22-debugging)

---

## 要不要升级？

| 情况 | 建议 |
|------|------|
| 你在用免费模型 | ✅ 建议升级 |
| 你有长时间运行会话 | ✅ 建议升级 |
| 你是桌面端用户 | ✅ 建议升级 |
| 用得好好的 | 可以等下次 |

**升级命令**：

```bash
opencode upgrade
```
